# Урок 3. Условия, циклы и методы: управляющая логика автотестов

## Что добавляется к базе

Теперь вы соединяете условия, циклы и методы в один рабочий сценарий: пройти по данным ответа, посчитать результаты и сделать итоговые проверки.

## 1) Цикл по данным

```java
int[] codes = {200, 201, 404, 500}; // Моделируем набор статус-кодов из нескольких вызовов.
int ok = 0;                          // Счётчик успешных проверок.

for (int code : codes) {             // Проходим по каждому коду без ручного дублирования.
    if (code >= 200 && code < 300) { // Проверяем правило успешности 2xx.
        ok++;                        // Увеличиваем счётчик успехов.
    }
}
```

## 2) Вынесение правила в метод

```java
static boolean is2xx(int code) {
    return code >= 200 && code < 300; // Одна переиспользуемая точка правды для проверки статуса.
}
```

Зачем:
- меньше копипаста,
- проще поддержка,
- легче писать понятные сообщения об ошибках.

## 3) Пример "как в реальном тесте"

```java
for (int code : codes) {                           // Идём по результатам вызовов.
    if (is2xx(code)) {                             // Используем переиспользуемый метод вместо дублирования условия.
        System.out.println("OK: " + code);        // Лог для успешной проверки.
    } else {
        System.out.println("FAIL: " + code);      // Лог для диагностики проблемного ответа.
    }
}
```
