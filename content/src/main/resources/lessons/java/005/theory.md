# Урок 5. Введение в автоматизированное тестирование: что и зачем автоматизируем

Автоматизируем не «всё подряд», а то, что:
- часто повторяется;
- критично для релиза;
- даёт быструю обратную связь;
- стабильно по требованиям.

## Приоритеты на старте
1. Smoke-кейсы (самые критичные пути).
2. Проверки API-контрактов (статус, обязательные поля).
3. Регрессии по ранее найденным багам.

## Что не автоматизировать сразу
- одноразовые проверки;
- нестабильный функционал, который постоянно переделывается;
- кейсы без понятного expected result.

Идея: автотесты — это инвестиция, они должны окупаться.
