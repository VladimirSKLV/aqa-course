# Урок 4. Advanced Java AQA: программа по блокам, урокам и обязательной практике

Ниже — расширенная структура advanced-трека: каждый важный раздел разбит на несколько уроков и имеет домашнее задание.

---

## Блок 1. Web UI: Selenium (отдельный блок)

### Урок 1. Поиск элементов в Selenium WebDriver
- `By.id`, `By.name`, `By.className`, `By.tagName`.
- Когда использовать стабильные атрибуты.

### Урок 2. CSS-локаторы
- Составные селекторы, атрибуты, псевдоклассы.
- Приоритет CSS для читаемости.

### Урок 3. XPath-локаторы
- Относительный XPath, оси, предикаты.
- Антипаттерны слишком «хрупких» XPath.

### Урок 4. Инфраструктура браузерных тестов
- Driver factory, конфиг, логирование, скриншоты, ретраи.

### Урок 5. Сложные UI-элементы и кастомные действия
- Drag&drop, hover, canvas, rich controls.

### Урок 6. Окна, вкладки, фреймы, файлы
- Переключение контекстов.
- Загрузка и скачивание файлов.

### Домашнее задание блока
1. Реализовать 10 UI-тестов на открытом онлайн-тренажёре.
2. Добавить тесты на вкладки/iframe/upload/download.
3. Сделать конфигурацию `baseUrl` для переключения на свой тренажёр.

## Блок 2. Web UI: Playwright (отдельный блок)

### Урок 1. Архитектура Playwright-проекта
- Контексты браузера, фикстуры, трейсинг.

### Урок 2. Локаторы и автождание
- `getByRole`, `getByTestId`, `locator(...)`.
- Принципы стабильных локаторов.

### Урок 3. Сложные UI-сценарии
- Multi-tab, iframe, диалоги, сетевые события.

### Урок 4. Запуск против онлайн-тренажёра
- Параметризация `baseUrl`.
- Запуск smoke/regression профилей.

### Домашнее задание блока
1. Подготовить 8 Playwright-тестов для публичного стенда.
2. Сравнить 3 сценария Selenium vs Playwright по стабильности.
3. Отчёт: что проще поддерживать и почему.

## Блок 3. API-автотестирование: Rest Assured (отдельный стек)

### Урок 1. Базовые запросы и валидации
- GET/POST/PUT/DELETE.
- Проверка статуса, заголовков, JSON-полей.

### Урок 2. Параметризация и тестовые данные
- Path/query params, data providers, динамические данные.

### Урок 3. Объекты с внутренним состоянием
- Test fixtures, изоляция данных.
- Контроль состояния между тестами.

### Урок 4. Исключения и негативные сценарии
- Тестирование 4xx/5xx.
- Проверка сообщений об ошибках.

### Домашнее задание блока
1. Написать 12 API-тестов (CRUD + негативные кейсы).
2. Добавить запуск на произвольном `baseUrl`.
3. Добавить из приложения кнопку/действие "Отправить тестовый запрос" на открытый API-тренажёр.

## Блок 4. Unit-архитектура и тестируемость

### Урок 1. Композиция и зависимости объектов
- DI без фреймворка и через фабрики.

### Урок 2. Mockito (отдельный урок)
- Mock, Spy, Stub, Verify.
- Изоляция внешних зависимостей.

### Урок 3. Наследование и расширяемость
- Когда наследование оправдано.
- Проблемы глубокой иерархии.

### Урок 4. Исключения и их тестирование
- `assertThrows`, проверка бизнес-ошибок.

### Урок 5. Lombok (отдельный урок)
- `@Getter/@Setter`, `@Builder`, `@Value`.
- Риски чрезмерного использования.

### Домашнее задание блока
1. Покрыть unit-тестами сервисный слой (минимум 80% line coverage).
2. Сделать минимум 5 тестов с Mockito.
3. Использовать Lombok в DTO-моделях и описать плюсы/минусы.
