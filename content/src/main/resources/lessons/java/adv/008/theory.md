# Урок 8. Структура папок и файлов

Начинаем новый блок: **Архитектура проекта + Gradle/Maven**.

Пример базовой структуры AQA-проекта:

- `src/test/java` — тесты;
- `src/main/java` — вспомогательный код (клиенты, утилиты);
- `src/test/resources` — конфиги, фикстуры, тестовые данные.

Рекомендуется выделять пакеты по ответственности:
- `api` (клиенты и модели);
- `tests` (наборы тестов);
- `config` (чтение параметров окружения);
- `utils` (вспомогательные функции).

Хорошая структура ускоряет онбординг и уменьшает хаос в больших наборах тестов.
