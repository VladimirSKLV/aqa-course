# Урок 1. Переменные и типы данных (Java)

## Почему это критично для AQA

Автотесты падают не только из-за "плохого Selenium", но и из-за базовой работы с данными:
- неверный тип для `id` или времени,
- потеря точности в числах,
- неучтённый `null`,
- ошибочные сравнения строк.

На этом уроке вы формируете фундамент, который влияет на стабильность всего тестового проекта.

## 1) Переменная и тип

Переменная — это именованное хранилище значения фиксированного типа.

```java
int userId = 123; // Храним id пользователя как целое число, потому что нам нужны арифметика и сравнения.
```

Почему важно:
- тип определяет допустимые значения,
- тип ограничивает операции,
- тип влияет на риск переполнения и на точность.

## 2) Основные типы, которые вы используете в тестах

### `int` и `long`

```java
int statusCode = 200;                 // HTTP-код всегда помещается в int.
long timestamp = System.currentTimeMillis(); // Время в миллисекундах хранится в long, иначе легко получить переполнение.
```

### `double` и точность

```java
double sum = 0.1 + 0.2; // Показывает проблему двоичного представления дробей.
System.out.println(sum); // Обычно увидите 0.30000000000000004, поэтому для денег так хранить нельзя.
```

Для денег берите `BigDecimal`.

### `boolean`

```java
boolean isActive = true; // Флаг состояния: активен ли пользователь/элемент/фича.
```

## 3) Целочисленное деление — частая ошибка

```java
int ratio = 5 / 2;      // Результат 2: дробная часть отбрасывается.
double exact = 5 / 2.0; // Результат 2.5: добавили вещественный операнд и получили точное деление.
```

В AQA это критично при проверках процентов и средних значений.

## 4) `String` и `null`

```java
String expected = "OK";                     // Эталонное значение из требований.
String actual = response.get("status");     // Значение из ответа API может быть null.
boolean same = expected.equals(actual);      // Сравниваем через equals, а не через ==.
```

Почему так:
- `==` у строк сравнивает ссылки, а не содержимое,
- `null` может вызвать `NullPointerException`, если вызвать метод у пустой ссылки.
