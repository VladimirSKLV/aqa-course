# Урок 4. Карта beginner-программы по Java AQA: блоки, уроки и домашние задания

В этом уроке мы фиксируем **полный подробный план** для Java Beginner с разделением на блоки и уроки.
Цель — чтобы в каждом важном блоке было несколько уроков, практика и понятное домашнее задание.

---

## Блок 1. База Java для автотестов

### Урок 1. Переменные, типы, строки и null
- Работа с `int`, `long`, `double`, `BigDecimal`.
- Почему `null` и сравнение строк (`equals`) критичны для тестов.

### Урок 2. Условия, циклы, функции
- `if/else`, `switch`, `for/while`, `break/continue`.
- Вынос повторяющихся шагов в методы: `login()`, `openProfile()`, `assertStatus()`.

### Урок 3. Основы ООП
- Классы и объекты.
- Инкапсуляция.
- Что такое объект с внутренним состоянием и как это влияет на тест.

### Домашнее задание блока
1. Сделайте мини-проект "валидация ответа API":
   - 3 метода проверки (код, время ответа, поле `status`).
   - 1 класс-модель ответа.
2. Покройте 5 кейсов (успех, ошибка, пустое поле, null, неверный тип).
3. Описать в README, как состояние объекта влияет на результат теста.

## Блок 2. Введение в автоматизированное тестирование

### Урок 1. Что и зачем мы автоматизируем
- Пирамида тестирования.
- Где полезны UI, API и unit-уровни.

### Урок 2. Пишем первые тесты (JUnit 5)
- `@Test`, `@BeforeEach`, `@AfterEach`.
- Проверки через assertions.

### Урок 3. Циклы, параметризованные тесты и аннотации
- `@ParameterizedTest`, `@ValueSource`, `@CsvSource`.
- Как избегать дублирования сценариев.

### Домашнее задание блока
1. Реализуйте 8 JUnit-тестов:
   - 3 обычных,
   - 3 параметризованных,
   - 2 негативных.
2. Добавьте отчёт в консоль о пройденных/упавших сценариях.
3. Зафиксируйте 2 правила именования тестов в проекте.

## Блок 3. Архитектура проекта + Gradle/Maven

### Урок 1. Структура папок и файлов
- Разделение `tests`, `pages`, `api`, `models`, `utils`, `config`.
- Паттерн "одна ответственность на пакет".

### Урок 2. Gradle/Maven в AQA-проекте
- Подключение зависимостей JUnit/Selenium/Rest Assured.
- Профили и системные параметры.

### Урок 3. Конфигурация BaseUrl
- Хранение в `application.properties`, `.env` или `-DbaseUrl=...`.
- Возможность подменить `BaseUrl` для своего сайта/тренажёра.

### Домашнее задание блока
1. Создать каркас проекта с каталогами `api`, `ui`, `core`, `config`.
2. Добавить запуск с параметром `-DbaseUrl=https://example.org`.
3. Реализовать проверку, что тесты падают с понятной ошибкой при пустом `baseUrl`.

## Блок 4. Первые интеграции: отправка тестовых запросов

### Урок 1. Отправка HTTP-запроса из приложения
- GET/POST на публичный тренажёр (например, reqres.in, jsonplaceholder).
- Логирование URL, кода ответа, тела.

### Урок 2. Валидация ответа
- Проверка обязательных полей.
- Базовая схема ошибок.

### Урок 3. Подготовка к API-автоматизации
- Почему этот шаг нужен перед Rest Assured.
- Как отделить transport-слой от проверок.

### Домашнее задание блока
1. Сделать сервис `ApiClient` с методом `sendTestRequest(baseUrl, endpoint)`.
2. Добавить 4 проверки:
   - код ответа,
   - время ответа,
   - наличие ключа,
   - негативный сценарий 404/400.
3. Добавить возможность смены `BaseUrl` без перекомпиляции.
