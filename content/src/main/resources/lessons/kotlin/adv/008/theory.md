# Kotlin Advanced 008. CI/CD и запуск API-автотестов из приложения

## Цель урока
Освоить подходы, которые используются в production-AQA: масштабируемая архитектура, отдельный API-блок, управляемые запуски и интеграция с CI.

## Теория
- **Изолируйте слои**: UI, API, data, util, fixtures.
- **Стандартизируйте проверки**: статус, schema/контракт, бизнес-правила, негативные сценарии.
- **Делайте запуски воспроизводимыми**: фиксированные входные данные, понятные параметры запуска (`baseUrl`, токены, окружение).
- **Интегрируйте с приложением и CI**: быстрый smoke из GUI + полноценный прогон в пайплайне.

## Пример реального сценария
1. QA выбирает тестовый сайт/стенд.
2. Запускает API smoke-suite из приложения.
3. По результатам решает: продолжать UI-регрессию или блокировать релиз.
4. Полный набор тестов уходит в nightly pipeline.

## Пример кода
```kotlin
fun shouldBeSuccessful(status: Int) {
    check(status in 200..299) { "Ожидали 2xx, получили $status" }
}

fun smokeReport(total: Int, passed: Int): String {
    val failed = total - passed
    return "total=$total passed=$passed failed=$failed"
}
```

## Где это применяется
- Предрелизные проверки интеграций.
- Быстрые sanity/smoke прогоны на открытых API-тренажёрах.
- Обучение junior-инженеров единому подходу к API-тестированию.
