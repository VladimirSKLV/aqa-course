# Урок 4. Kotlin Beginner AQA: подробная программа по блокам и урокам

Этот урок закрепляет расширенную структуру курса: в каждом блоке несколько уроков и обязательное домашнее задание.

---

## Блок 1. Kotlin-основа для автотестов

### Урок 1. Типы, nullable и безопасные вызовы
- `String?`, `?.`, `?:`, `requireNotNull`.
- Защита тестов от падений из-за `null`.

### Урок 2. Функции и лямбды
- `fun`, параметры по умолчанию, именованные аргументы.
- Extension-функции для тестовых шагов.

### Урок 3. Основы ООП
- Классы, data class, инкапсуляция.
- Объекты с внутренним состоянием в тестовом контексте.

### Домашнее задание блока
1. Создать data class для API-ответа.
2. Написать 4 функции-валидатора и 2 extension-функции.
3. Продемонстрировать обработку nullable-полей в позитивном и негативном кейсе.

---

## Блок 2. Введение в автотестирование

### Урок 1. Пишем первые тесты на JUnit 5
- `@Test`, lifecycle-аннотации.
- Базовые assertions.

### Урок 2. Циклы, параметризованные тесты и аннотации
- `@ParameterizedTest` + `@MethodSource`/`@CsvSource`.
- Повторяемые наборы данных.

### Урок 3. Исключения и тестирование ошибок
- `assertThrows`, проверка текста исключения.

### Домашнее задание блока
1. Реализовать 7 тестов: 4 позитивных, 3 негативных.
2. Добавить минимум 2 параметризованных сценария.
3. Вынести общий setup в отдельные функции/фикстуры.

---

## Блок 3. Архитектура + Gradle/Maven

### Урок 1. Структура каталогов
- `tests/ui`, `tests/api`, `pages`, `clients`, `models`, `config`.

### Урок 2. Управление зависимостями
- Подключение библиотек, версии, test tasks.

### Урок 3. Подмена BaseUrl
- `gradle.properties`, `-DbaseUrl`, env-переменные.
- Поддержка пользовательского тренажёра или своего сайта.

### Домашнее задание блока
1. Собрать проект с отдельными пакетами для UI и API.
2. Добавить запуск тестов с `-DbaseUrl`.
3. Реализовать fail-fast при некорректном URL.

---

## Блок 4. Первые интеграции с онлайн-тренажёром

### Урок 1. Отправка тестового HTTP-запроса из приложения
- Запрос к открытому API.
- Логирование параметров и результатов.

### Урок 2. Базовая проверка ответа
- Код ответа, обязательные поля, время.

### Урок 3. Подготовка к Rest Assured
- Выделение слоя клиента и слоя проверок.

### Домашнее задание блока
1. Сделать `TestApiClient` с настройкой `baseUrl`.
2. Написать 4 проверки ответа.
3. Добавить возможность быстро сменить endpoint под ваш тренажёр.

## Результат
После урока у вас есть полноценный beginner-план с детализацией, практикой и готовностью к advanced-блокам Selenium/Playwright/API.
